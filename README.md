# Unity_TeamPJ

신동혁 신승환 유니티프로젝트 분업 브랜치작업공간

---

## 팀 구성

| 이름 | 브랜치 | 폴더 | 담당 예시 |
|------|--------|------|-----------|
| 신동혁 | SDH | Assets/2.SDH | PLAYER |
| 신승환 | SSH | Assets/1.SSH | ENEMY |

---

## 브랜치 구조

```
main        ← 최종 결합본 (완성된 것만 올라옴)
├── SDH     ← 신동혁 작업 브랜치
└── SSH     ← 신승환 작업 브랜치
```

---

## 작업 흐름

### 1. 각자 브랜치에서 작업
각자 자신의 브랜치에서만 작업해요.
- 신동혁 → SDH 브랜치에서 PLAYER 관련 작업
- 신승환 → SSH 브랜치에서 ENEMY 관련 작업

### 2. 작업 완료 후 GitHub에서 PR 생성
```
SDH → main  (신동혁 PR)
SSH → main  (신승환 PR)
```
GitHub에서 Compare & Pull Request 버튼 클릭
→ 제목 작성 후 Create Pull Request

### 3. 상대방이 검토 후 Approve → Merge
상대방이 내용 확인 후 승인하면 main에 병합돼요.

### 4. main에서 유니티 실행 테스트
병합된 결과물을 main 기준으로 실행해서 확인해요.

## 오류 발생 시 대처법

### 케이스 1 - 수정이 빠를 때 (대부분의 경우)
```
main에서 오류 발견
      ↓
해당 브랜치(SDH 또는 SSH)에서 수정 후 push
      ↓
다시 PR → Merge
      ↓
정상 작동 확인
```

### 케이스 2 - 수정이 오래 걸릴 때
```
main에서 오류 발견
      ↓
문제된 Merge 되돌리기 (main 이전 상태로 복구)
      ↓
해당 브랜치에서 수정 완료 후 다시 PR → Merge
```

---

## 주의사항

- 상대방 브랜치에는 절대 push하지 않기
- 작업 전 항상 자기 브랜치 확인
  ```
  git branch   ← * 표시가 자기 브랜치에 있어야 함
  ```
- main에 직접 push 금지, 반드시 PR로만 병합
