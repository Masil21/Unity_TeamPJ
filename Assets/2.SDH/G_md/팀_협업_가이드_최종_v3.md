# 슈팅 게임 팀 협업 가이드 (v5 - 2026-05-05 최종 반영)

## 팀원 정보
- **신동혁 (SDH 브랜치)** — Enemy, Item, UI배치 파트
- **신승환 (SSH 브랜치)** — Player, 총알, 배경, UI관리 파트

## 프로젝트 개요
- Unity 2D 세로형 슈팅 게임
- 해상도: 1920x1080 Portrait
- 프레임: 60fps 고정
- 코드 패턴: **싱글톤, 대리자(Action), 코루틴** 활용

## Git 브랜치 구조
```
main (통합용, merge 대상)
 ├─ SSH (신승환 브랜치)
 └─ SDH (신동혁 브랜치)
```
- 각자 브랜치에서 작업 → main에 merge
- merge 후 상대방은 main에서 pull 받아서 최신 상태 유지
- **같은 씬 파일(.unity)은 동시에 수정하지 않기**
- **UIManager.cs는 신승환이 전부 관리** (한 사람이 관리해야 충돌 방지)
- **Item3.cs는 Assets/2.SDH/Scripts/ 하나만 존재** (1.SSH 폴더에 복사 금지 — CS0101 에러)

### 사용 패턴 (싱글톤 / 대리자 / 코루틴)

**싱글톤:**
```csharp
ItemManager3.Instance.CreateItem(pos);  // 아이템 생성
UIManager.Instance.AddScore(100);       // 점수 추가 (UIManager는 SSH 관리)
```

**대리자 (Action):**
```csharp
// SpawnPoint.cs — 적 생성 시 등록
ec.onDie = ItemManager3.Instance.CreateItem;

// EnemyController.cs — 사망 시 실행
onDie?.Invoke(transform.position);
```

**코루틴:**
```csharp
// EnemyBullit.cs — OnEnable()에서 시작 (풀에서 재활성화 시 자동 재시작)
void OnEnable() { StartCoroutine(FireRoutine()); }

// Item3.cs — 아이템 낙하
IEnumerator Move() { ... }

// BoomEffect.cs — 2초간 적/적총알 파괴
IEnumerator BoomRoutine() { ... }
```

### 대리자 흐름 (아이템 드랍)
```
SpawnPoint → 적 Instantiate
           → ec.onDie = ItemManager3.Instance.CreateItem  // 등록

적 피격 → hp <= 0 → isDead = true → onDie?.Invoke(pos)  // 실행
                  → ItemManager3.CreateItem(pos)
                  → 코루틴으로 아이템 낙하
```

> ⚠️ **isDead 플래그 필수** — 여러 총알이 동시에 적을 맞혀 onDie가 2번 실행되는 것을 방지

### EnemyBullit.cs 주의사항
- `Start()` 대신 **`OnEnable()`** 에서 FireRoutine 시작
- 오브젝트 풀로 `SetActive(false) → true` 재활성화 시 코루틴 자동 재시작됨

### 적 SpawnPoint 배치
- Point 0~4: 위쪽에서 아래로 (Vector3.down)
- Point 5~6: 오른쪽에서 왼쪽으로 (Vector3.left)
- Point 7~8: 왼쪽에서 오른쪽으로 (Vector3.right)
- Point 9→10: 대각선 (방향벡터 계산)
- Point 11→12: 대각선 (방향벡터 계산)
- Point 10, 12는 목표 방향용 (적 생성 안 함)

### 아이템 드랍 확률
- None: 30%
- Coin: 30%
- Power: 20%
- Boom: 20%

### Boom 이펙트 흐름
```
Player 우클릭
  → UIManager.Instance.UseBoom() → boomCount 차감 → true 반환
  → Player.cs에서 Instantiate(Boom_Boom.prefab, player.position)
  → BoomEffect.cs Start() → BoomRoutine() 코루틴 시작
  → 2초간 매 프레임: 태그 "Enemy" 전체 TakeDamage(9999) + 태그 "EnemyBullet" 전체 Destroy
  → 2초 후 Destroy(gameObject)
```

### Player.cs 현재 상태 (완료)

**주요 필드:**
```csharp
public float speed = 5f;
public int hp = 3;           // 목숨 수 (게임오버와 별도로 Player 자체 HP)
public int bulletDamage = 5;
public float fireInterval = 0.2f;
public Transform bulletSpawnOffset;
public GameObject sideBulletPrefab;
public GameObject centerBulletPrefab;
public GameObject boomAnimationPrefab;  // Inspector에 Boom_Boom 프리팹 연결됨
public int power = 1;        // 시작: 1단계
public float sideOffset = 0.25f;
[SerializeField] float respawnDelay = 1f;
```

**HP / 리스폰 흐름:**
```
적 총알 충돌 → Player.TakeDamage()
  → _isInvincible이면 무시
  → hp--
  → hp > 0: InvincibleRoutine(1.5초 무적)
  → hp <= 0: hp = _maxHp, power = 1, RespawnRoutine()
               SpriteRenderer.enabled = false (1초 대기) → 시작 위치 이동
               SpriteRenderer.enabled = true (2초 무적 후 해제)
```

> ⚠️ **RespawnRoutine은 SpriteRenderer.enabled 방식** — `gameObject.SetActive(false)` 쓰면 코루틴 멈춰서 리스폰 안 됨

**총알 단계:**
- power=1: 정면 1발
- power=2: 양옆 2발
- power=3: 정면(강화) + 양옆 2발
- 피격 시 power 유지, **사망(hp=0) 시에만** power=1 리셋

**Boom 우클릭:**
```csharp
// UIManager 있을 때: UIManager.UseBoom() → boomCount 차감
// UIManager 없을 때: _boomCount 로컬 변수로 폴백 처리
if (Input.GetMouseButtonUp(1))
{
    bool used = UIManager.Instance != null
        ? UIManager.Instance.UseBoom()
        : (_boomCount > 0 && --_boomCount >= 0);
    if (used && boomAnimationPrefab != null)
        Instantiate(boomAnimationPrefab, transform.position, Quaternion.identity);
}
```

**아이템 획득 (OnTriggerEnter2D):**
```csharp
// UIManager가 없어도 아이템은 획득됨. 점수만 UIManager 있을 때 추가
case Item3.ItemType.Coin:  if (UIManager.Instance != null) UIManager.Instance.AddScore(100);
case Item3.ItemType.Power: PowerUp(); if (UIManager.Instance != null) UIManager.Instance.AddScore(200);
case Item3.ItemType.Boom:  AddBoom(); if (UIManager.Instance != null) UIManager.Instance.AddScore(300);
```

### UIManager.cs 현재 상태 (완료)

**싱글톤 구조:**
```csharp
public static UIManager Instance;
void Awake() { if (Instance != null && Instance != this) { Destroy(gameObject); return; } Instance = this; }
```

**완료된 기능:**
- `images[]` — 목숨 아이콘 (Inspector에서 Life_1, Life_2, Life_3 Image 연결)
- `boomImages[]` — Boom 아이콘 (Inspector에서 Boom_1, Boom_2, Boom_3 Image 연결)
- `gameOverPanel` — 게임오버 패널
- `retryButton` — Retry 버튼 (클릭 시 현재 씬 재로드)
- `scoreText` — 점수 TextMeshPro
- `finalScoreText` — 게임오버 시 최종점수 표시

**핵심 공개 API:**
```csharp
UIManager.Instance.AddScore(int amount)                         // 점수 추가
UIManager.Instance.DecreaseLife()                               // 목숨 1 감소, 0이면 GameOver (bool 반환)
UIManager.Instance.HandlePlayerHit(go, respawnPos, delay)       // 피격→비활성→리스폰 코루틴
UIManager.Instance.AddBoom()                                    // Boom 카운트 +1 (최대 3)
UIManager.Instance.UseBoom()                                    // Boom 사용, bool 반환
```

> ⚠️ **Player.cs는 UIManager.HandlePlayerHit을 호출하지 않음**
> Player가 자체 HP를 관리하고 SpriteRenderer로 리스폰 처리.
> HandlePlayerHit은 외부에서 별도로 목숨 UI 연동이 필요할 때 사용.

### PlayerBullet.cs 현재 상태 (완료)
```csharp
void OnTriggerEnter2D(Collider2D other)
{
    if (other.CompareTag("Enemy"))
    {
        EnemyController ec = other.GetComponent<EnemyController>();
        if (ec != null) ec.TakeDamage(damage);  // SendMessage 대신 직접 호출
        Destroy(gameObject);
    }
}
```

| 제공 | 내용 |
|------|------|
| Player.TakeDamage() | 적 총알 충돌 시 동혁이 호출 |
| Player.PowerUp() | Power 아이템 획득 시 Player 내부에서 자동 호출 |

### 신동혁이 신승환 코드를 호출하는 코드 예시
```csharp
// 적 총알이 Player 맞혔을 때 (EnemyBullitController.cs)
Player player = other.GetComponent<Player>();
if (player != null)
{
    player.TakeDamage();
    Destroy(gameObject);
}
```

- Circle Collider 2D (Is Trigger ✓)

- Capsule Collider 2D (Is Trigger ✓)
- Rigidbody2D (Gravity Scale 0, Body Type Kinematic)
- Player.cs
