# Unity_TeamPJ
신동혁 유니티프로젝트 분업 브랜치작업공간

---

## 브랜치 병합 흐름

```
main        ← 최종 결합본
├── SDH     ← 신동혁 작업 브랜치 (현재)
└── SSH     ← 신승환 작업 브랜치
```

SSH, SDH 브랜치에서 각자 작업 후 main에 PR → Merge 하는 방식으로 진행합니다.

---

## main 병합 후 오류 발생 시

Git은 파일 상태가 아닌 **변경 이력(commit)** 기준으로 병합합니다.
main에 오류 코드가 들어있어도 해당 브랜치에서 수정 후 다시 merge하면 수정본으로 업데이트됩니다.
main을 직접 수정할 필요 없습니다.

### 경우의 수

**SDH(나)만 문제인 경우**
```
SDH 브랜치에서 수정 → commit → PR → main merge
(SSH는 건드릴 필요 없음)
```

**SSH(상대방)만 문제인 경우**
```
SSH 브랜치에서 수정 → commit → PR → main merge
(SDH는 건드릴 필요 없음)
```

**SSH + SDH 둘 다 문제인 경우**
```
각자 자기 브랜치에서 수정 → commit → 각각 PR → main merge 2번
(순서는 누가 먼저든 상관없음)
```

---

## 씬/프리팹 충돌 주의

유니티 `.unity` `.prefab` 파일은 Git이 자동으로 합치지 못합니다.

```
SDH가 같은 씬 파일 수정
SSH도 같은 씬 파일 수정
→ main merge 시 충돌(conflict) 발생
→ 수동으로 어느 쪽을 살릴지 결정 필요
```

충돌을 줄이려면 **서로 다른 씬/프리팹을 담당하거나, 같은 씬은 한 명씩 번갈아 작업**합니다.

---

## 정리

| 상황 | 해결법 |
|------|--------|
| 로직/코드 오류 | 해당 브랜치에서 수정 후 다시 merge |
| 씬/프리팹 충돌 | merge 시 수동으로 충돌 해결 |
| main 직접 수정 | 거의 불필요, 하면 오히려 복잡해짐 |
